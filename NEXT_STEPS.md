# Next Steps

## 在另一台电脑继续工作

1. 登录 Codex，并连接 GitHub connector。
2. 打开仓库 https://github.com/tya34/Codex-。
3. 下载或复制 `config.toml` 中的 `developer_instructions` 到新电脑的 Codex 全局配置。
4. 将 `memories/download-cleanup.md` 和 `memories/work-upload-command.md` 放入新电脑的 Codex 全局记忆目录，或让 Codex 读取这两个文件后按其规则工作。
5. 继续工作前，让 Codex 先读取 `README.md`、`WORKLOG.md` 和本文件，恢复当前上下文。

## 之后建议

- 每次完成重要工作后，说“把这次工作整理上传”。
- 让 Codex 只上传最终文件和整理后的记录，不上传临时文件、缓存、草稿、日志或中间产物。
- 如果某次工作涉及科研脚本、FDTD 模型、图片或论文材料，建议单独建项目仓库，并在项目仓库中维护 `README.md`、`WORKLOG.md`、`NEXT_STEPS.md` 和最终输出文件。

## 注意

- 这个仓库保存的是整理后的工作交接记录，不等同于恢复原始聊天线程。
- 原始对话中可能包含本机路径、账号状态或临时调试信息，默认不上传完整原始聊天日志。
- 如果确实需要完整对话记录，应先明确要求，并在上传前检查是否包含敏感信息。
